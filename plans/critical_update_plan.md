# План доработки дашборда: фокус на Кубарева Михаила + все месяцы

## 1. Текущее состояние vs желаемое

| Что есть сейчас | Что нужно |
|---|---|
| Парсер принимает tab-separated текст из textarea | ✅ Оставить, это удобно для копипасты из Excel |
| Дефолтные данные: только июнь 2025 | ❌ Нужны ВСЕ месяцы (июнь25 – апрель26) |
| Анализируются ВСЕ менеджеры | ❌ Только Кубарев Михаил |
| Нет разбивки по месяцам | ❌ Нужен тренд по месяцам |
| 2 графика (Donut + Stacked Bar) | Нужен 3-й график: динамика по месяцам |

---

## 2. Что нужно сделать (пошагово)

### Шаг 1: Извлечь все данные Кубарева Михаила из Excel
- В файле 11 месячных вкладок: `июнь 2025` ... `апрель 2026`
- В каждой вкладке найти строки, где `Исполнитель = "Кубарев Михаил"` (и строки ниже с пустым исполнителем, пока не встретится другой менеджер)
- Собрать сводный массив: `{ month, ...статусы }`
- Сгенерировать единый tab-separated текст для предзагрузки в textarea

### Шаг 2: Обновить парсер
- Добавить поддержку колонки **месяц** (сейчас её нет в формате)
- Новый формат строки: `Месяц \t Исполнитель \t Категория \t ... 8 колонок цифр`
- Или: парсить несколько «блоков» разделённых заголовком месяца

### Шаг 3: Добавить фильтрацию — только Кубарев Михаил
- После парсинга отфильтровать `rows.filter(r => r.manager === "Кубарев Михаил")`
- Или встроить фильтр прямо в парсер

### Шаг 4: Новый график — Динамика по месяцам (Line Chart)
- Ось X: месяцы (июнь25, июль25, ..., апрель26)
- Линии: Заказы, Сделка сорвалась, Не прошли по цене, Прочие потери
- Это покажет тренд: растёт ли конверсия, падает ли % отказов

### Шаг 5: Обновить KPI-карточки
- Показывать статистику **только по Кубареву**
- Добавить карточку: «Месяцев данных» (11)

### Шаг 6: Адаптировать умные подсказки
- Триггеры должны работать в контексте одного исполнителя
- Добавить 4-й триггер: «Динамика ухудшается» — если конверсия падает 2+ месяца подряд

### Шаг 7: Предзагрузка всех данных
- Node.js скрипт (`scripts/extract_kubarev.js`) вытащит все строки Кубарева из Excel
- Результат сохранить в `src/defaultData.js` как строку-константу
- Dashboard импортирует эту строку как `DEFAULT_TEXT`

## 3. Что ты упустил из критического (я добавил)

| Упущено | Почему критично |
|---|---|
| **Динамика по месяцам** | Без неё ты не видишь тренд: становится лучше или хуже. Самое важное для управления |
| **Категории запросов** | У Кубарева может быть не только «прямое ж/д», но и другие категории. Нужна разбивка |
| **Конверсия по месяцам (линия)** | Показывает, когда и почему проседала конверсия |
| **Автоматическая выгрузка из Excel** | Копипастить 11 месяцев вручную — ад. Нужен скрипт, который сам соберёт все вкладки |
| **Фильтр по категории** | Возможность смотреть только «прямое ж/д» отдельно от остального |
| **Сравнение с другими менеджерами** | Если смотреть только Кубарева, ты теряешь контекст. Может, оставить галочку «Показать всех»? |

## 4. Новая структура компонентов

```
Dashboard.jsx
├── Textarea (импорт данных) + кнопка «Анализировать»
├── Строка KPI (5 карточек):
│   ├── Всего запросов (Кубарев)
│   ├── Конверсия в заказ %
│   ├── Мусорные запросы %
│   ├── Среднее в месяц
│   └── Месяцев данных
├── Графики (3 шт, сетка 2+1):
│   ├── [Row 1] Donut: Причины потерь (Кубарев)
│   ├── [Row 1] Stacked Bar: Категории запросов (Кубарев)
│   └── [Row 2] Line Chart: Динамика по месяцам (Заказы / Отказы / Потери)
└── Умные подсказки (3-4 карточки)
```

---

## 5. Порядок реализации

1. Написать `scripts/extract_kubarev.js` — вытащить все строки Кубарева из Excel в один текстовый блок
2. Сгенерировать `src/defaultData.js` с полным текстом всех 11 месяцев
3. Дописать парсер: поддержка маркера месяца в данных
4. Добавить Line Chart (Recharts) для динамики по месяцам
5. Обновить KPI, графики и инсайты под фильтр «только Кубарев»
6. Протестировать на всех 11 месяцах
